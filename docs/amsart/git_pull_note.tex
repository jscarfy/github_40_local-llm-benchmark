\title{A Fast-Forward Only Pull Wrapper}
\author{}
\maketitle

\section{Statement}
We define a repository synchronization procedure that performs a pull using fast-forward only, and repairs missing upstream tracking information.

\begin{definition}
The \emph{pull wrapper} is the shell procedure:
\begin{verbatim}
./scripts/pull_ff_only.sh
\end{verbatim}
\end{definition}

\begin{theorem}
If the current branch has a corresponding remote branch, then the wrapper configures upstream (if missing) and updates the working tree using a fast-forward-only pull.
\end{theorem}

\begin{proof}
The script first ensures it runs at the repository root, then checks for an upstream reference. If missing, it sets upstream to \texttt{remote/branch} (or pushes to create it). Finally, \texttt{git pull --ff-only} either fast-forwards or fails without creating merge commits.
\end{proof}
